# Example: A Good Pull Request Description

Below is an example of what a well-written PR looks like for this workshop.

---

## PR Title

> Add Kyle to participants list

## PR Body

> ### What changed
> - Added my entry to `participants.md` with my name, GitHub handle, and learning goal
>
> ### Why
> Completing the workshop exercise to practice the branch-commit-PR workflow.
>
> ### How I tested
> - Previewed the markdown table in GitHub to make sure formatting is correct
> - Verified my row appears in the right place in the table
>
> ### Checklist
> - [x] Created a branch (not committing directly to main)
> - [x] Commit message is descriptive
> - [x] PR description is filled out
> - [x] Markdown table renders correctly

---

## Why This Is Good

- **Title** is short and describes the change
- **What changed** tells the reviewer exactly what to look for
- **Why** gives context — even for a simple change
- **How I tested** shows you verified your own work
- **Checklist** confirms you followed the steps
